Authentication Approach

This is a production platform handling cold chain logistics — auth affects every API call, agent execution, and admin action

A

JWT + Roles (RBAC)

Username/password login with JWT tokens. Roles: Admin, Operations Manager, Analyst, Read-Only. API key support for machine-to-machine (IoT agents, MCP servers).

Pros

  • Already proven in NEXUS + Legal AI
  • Self-contained, no external deps
  • Fast to implement
  • API keys for IoT integrations

Cons

  • No SSO for enterprise clients
  • Password management burden
B

JWT + OAuth2 SSO (Enterprise)

Login via Microsoft Entra ID or Google Workspace in addition to local accounts. Enterprise clients plug in their existing identity provider — no separate passwords. API keys for service accounts.

Pros

  • Enterprise-standard for Lineage-scale clients
  • No password fatigue
  • Centralized user management for client

Cons

  • More setup (OAuth app registration)
  • Extra complexity for demo environments
C

JWT + OAuth2 + API Keys (Full Stack)

All three: local JWT for demo/dev, OAuth2 SSO for enterprise production users, and API keys for IoT devices and service-to-service calls. Feature-flagged — SSO is opt-in per deployment.

Pros

  • Covers every use case
  • Demo works without OAuth setup
  • Enterprise-ready when client approves

Cons

  • Most complex auth layer
  • Requires careful testing of all paths

Recommendation

Option C — implement JWT + RBAC fully now (you can demo it immediately), design the OAuth2 layer as opt-in config so enterprise clients can plug in Microsoft/Google when they approve the deployment. API keys are essential for IoT device auth.
